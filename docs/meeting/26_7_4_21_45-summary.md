## 中心アイデア: agent-first なチャット/コラボレーションシステム(「Slack を AI-first で作り直す」)

- 今日の課題は「AI に何ができるか」ではなく「会社が本当に何をしたいか」。全社のアラインメントは実際には Slack で起きている。**人間に残る場所はチャットレイヤーだけ**になり、実行はエージェントがやる。
- 既存の姿(Claude Code を Slack に繋ぐ等)は、エージェントが「ただの1メンバー」として参加するだけ。Notion・Google 等ツールを大量に接続する手間もある。**チャットシステムそのものを agent-native に設計し直す**のが余白。
- 3つの RFS テーマを1本で貫ける:
  - **Company Brain**: 会社のデータはそもそも Slack から始まる。契約書もダッシュボードも議論もチャネルに流れる——チャットがデータの起点であり、そこから brain を構築できる。
  - **Dynamic Software Interfaces**: エージェントの応答はメッセージ(テキスト/画像/動画)である必要はない。クライアント専用チャネルで「このクライアントの売上履歴のダッシュボードを見せて」と言えば、**チャネル内にその場で UI が生成される**。これが組織のインターフェースになる——今は誰もやっていない。
  - **Software for Agents**: このチャットシステム内でエージェントはネイティブな一級参加者。
- 多人数で効く: 1人なら既存ツールで足りる。**20人規模のチームでタスク委任・調整・エージェントセッションの共有**が起きるときに価値が出る(マルチプレイヤー)。スキルがボトルネックでなくなった今、全員がエージェントにタスクを出せる——その調整レイヤーがチャット。
- エージェントの**セッション自体をリアルタイムで共有・観戦**できる(Slack はメッセージ/動画/画像しか流せないが、進行中のセッションを流せる)。

## Core Idea: An Agent-First Chat/Collaboration System ("Rebuilding Slack, AI-First")

- Today's real question isn't "what can AI do" but "what does the company actually want to do." Company-wide alignment already happens in Slack. **The only place left for humans is the chat layer** — agents do the executing.
- The current approach (e.g., wiring Claude Code into Slack) just adds the agent as "one more member." It also takes a lot of work to connect a pile of tools like Notion and Google. The open space is to **redesign the chat system itself to be agent-native**.
- One build can thread all three RFS themes:
  - **Company Brain**: company data originates in Slack in the first place. Contracts, dashboards, and discussions all flow through channels — chat is the origin point of the data, and you can build the "brain" from there.
  - **Dynamic Software Interfaces**: an agent's response doesn't have to be a message (text/image/video). In a client-only channel, say "show me a dashboard of this client's sales history" and **a UI is generated on the spot, right inside the channel**. That becomes the organization's interface — nobody is doing this today.
  - **Software for Agents**: inside this chat system, agents are native, first-class participants.
- This matters more at scale: for a single person, existing tools are enough. The value shows up when a **20-person team is delegating tasks, coordinating, and sharing agent sessions** (multiplayer). Now that skill is no longer the bottleneck, everyone can hand tasks to an agent — and chat is the coordination layer for that.
- **Agent sessions themselves can be shared and watched live** (Slack can only carry messages/video/images, but here you can stream an in-progress session).
